Creating an Object

Next >>

In JavaScript, every value is an object except primitive values. Primitive values are values that have no properties or methods. They are usually hard coded and cannot be changed. If x = 3.14, you can change the value of x. But you cannot change the value of 3.14. With that in mind, this means that objects are variables but objects can contain many values. This is where object literals come in and are defined as comma-separated lists of name-value pairs.

Code:

  1. /*
  2. * Using object literals applies some best practice concepts where it prevents naming confilts of variables that are defined globaly. This provides a much cleaner
  3. * approach to assigning and accessing values.
  4. */
  5. var app = {
  6.     version: 1.0,
  7.     name: 'Vanilla JS Revision',
  8.     getVersion: function () {
  9.         /*
  10.         * In below context, 'this' refers to the owner object 'app'. It gives us the ability to access the different values in the object literal.
  11.         * More info: https://www.w3schools.com/js/js_this.asp
  12.         */
  13.         return this.version;
  14.     }
  15. };
  16.  
  17. /*
  18. * To access a value in an object literal, we use the dot notation with the variable name like so.
  19. */
  20. console.log('Object property result: ' + app.name);
  21.  
  22. /*
  23. * We can also access methods within the object literal using the dot notation and adding brackets in the end of the declaration.
  24. */
  25. console.log('Object method result: ' + app.getVersion());
